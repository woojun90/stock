### 3.2 为什么选择AI Agent驱动

| 维度 | 传统代码方案 | AI Agent + Skill方案 |
|-----|------------|---------------------|
| **灵活性** | 规则硬编码，改规则需改代码 | 规则配置化，AI动态解读 |
| **适应性** | 政策市、黑天鹅需人工干预 | AI可主动识别异常并调整 |
| **可解释性** | 只有代码逻辑 | AI可输出决策理由 |
| **维护成本** | 代码量大，维护复杂 | Skills独立，规则可热更新 |
| **扩展性** | 新功能需写代码 | 新增Skill即可扩展能力 |

### 4.2 Skill: momentum-analyzer (动量分析)

```
┌─────────────────────────────────────────────────────────────────────────┐
│ Skill: momentum-analyzer                                                │
├─────────────────────────────────────────────────────────────────────────┤
│ 触发: 调仓日前 / 手动调用                                               │
│ 输入: ETF历史数据、动量周期(默认21日)                                    │
│ 输出:                                                                   │
│   - 动量排名表 (Z-Score标准化)                                          │
│   - 趋势判断 (50日均线方向)                                             │
│   - 推荐买入列表 (前20% + 趋势向上)                                     │
│ 能力:                                                                   │
│   - 计算N日收益率                                                       │
│   - Z-Score标准化排序                                                   │
│   - MA50趋势过滤                                                        │
│   - 生成结构化信号报告                                                  │
└─────────────────────────────────────────────────────────────────────────┘
```

**核心算法:**
```python
# 21日动量计算
momentum_21 = (price[t] - price[t-21]) / price[t-21]

# Z-Score标准化
z_score = (momentum - μ) / σ

# 趋势过滤
trend_ok = price > MA50

# 信号生成
signal = z_score > 1.0 AND trend_ok
```
